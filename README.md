# Kira Mind Bot

Персональный Telegram-ассистент с долговременной памятью, проактивными подсказками и веб-панелью управления. Разворачивается на своём VPS одной командой — никаких подписок, никакого чужого облака.

---

## Быстрый старт

> Нужно: VPS с Ubuntu/Debian, SSH-доступ по ключу, Node.js 20+ на локальной машине.

```bash
git clone <repo>
cd kira-mind-bot
./install.sh --server-ip YOUR_VPS_IP
```

Скрипт сам установит Docker на сервер, соберёт все ключи в диалоге и задеплоит бота. В конце выведет ссылку и пароль к панели управления.

---

## Веб-панель управления

После деплоя открывается по ссылке вида `http://YOUR_VPS_IP:PORT` — порт генерируется автоматически и сохраняется между деплоями.

![Admin Panel](.github/admin-preview.png)

**Что можно делать из браузера без правки файлов:**

- Менять все API-ключи, токены и параметры бота
- Настраивать расписание проактивных сообщений (с удобным выбором часов/минут/дней вместо миллисекунд)
- Редактировать личность бота: системный промпт, биографию, стиль общения, варианты настроения
- Перезапускать контейнеры после изменений (кнопка рестарта или чекбокс «рестарт» при сохранении)
- Видеть статус контейнеров и время работы в реальном времени

Учётные данные для входа выводятся в конце каждого деплоя и хранятся на сервере в `/root/.kira-admin-state`.

## Что умеет бот

### Основные сценарии

- Свободный диалог с памятью о тебе — бот помнит факты между разговорами
- Напоминания: создание, перенос, завершение, просмотр активных
- Голосовые сообщения: распознавание речи через OpenAI Whisper
- Изображения: анализ фото, OCR, генерация картинок по запросу (Ideogram)
- Геолокация и карты: маршруты, поиск мест рядом, данные по адресу
- Веб-поиск: актуальная информация через OpenAI web search
- Пересланные треды: объединение и разбор в контексте

### Работа с твоим Telegram-аккаунтом

Если настроить Telegram User Client (`TELEGRAM_API_ID`, `TELEGRAM_API_HASH`, `TELEGRAM_SESSION_STRING`):

- Сводка непрочитанных сообщений за нужный период
- Анализ переписки с конкретным контактом или группой
- Отправка сообщений и отложенная отправка от твоего имени
- Режим переговоров — бот уточняет позицию и ведёт диалог за тебя

### Память и проактивность

- Долговременная векторная память по доменам: работа, здоровье, семья, путешествия и др.
- Факты извлекаются из диалогов и используются в следующих ответах
- Три фоновых планировщика:
  - **kiraLifeScheduler** — бот пишет первым, как живой человек
  - **memoryInsightScheduler** — подсказывает о планах в нужный момент (пятница вечером → напоминание про бар на выходных)
  - **dmReportScheduler** — периодические дайджесты входящих сообщений
- Все фоновые сообщения идут только в личку, не утекают в группы

### Групповые чаты

- Реагирует только на прямые `@упоминания`
- `GROUP_PUBLIC_MODE=false` (по умолчанию): посторонним отвечает остроумной отшуткой, ничего личного не раскрывает
- `GROUP_PUBLIC_MODE=true`: отвечает всем участникам — диалог, карты, поиск, картинки — без доступа к личной памяти

## Установка шаг за шагом

### 1. Подготовка VPS

Подойдёт любой VPS с Ubuntu 20.04+ или Debian 11+. Минимальные характеристики: 1 CPU, 2 GB RAM, 20 GB диск.

Настрой SSH-доступ по ключу:

```bash
ssh-copy-id root@YOUR_VPS_IP
```

### 2. Что нужно заготовить заранее

**Обязательно:**
- OpenAI API Key — [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
- Токен Telegram-бота — создать через [@BotFather](https://t.me/BotFather) → `/newbot`
- Твой Telegram User ID — узнать через [@userinfobot](https://t.me/userinfobot)

**Опционально (для полной функциональности):**
- [Google Maps API Key](https://console.cloud.google.com) — маршруты и карты
- [Ideogram API Key](https://ideogram.ai/manage-api) — генерация изображений
- Telegram API ID + Hash — [my.telegram.org/apps](https://my.telegram.org/apps) — для чтения переписок

### 3. Запуск установщика

```bash
git clone <repo>
cd kira-mind-bot
./install.sh --server-ip YOUR_VPS_IP
```

Скрипт сделает всё сам:
1. Проверит SSH-соединение
2. Установит Docker на VPS (если не установлен)
3. Интерактивно спросит все ключи с подсказками где их взять
4. Сгенерирует `.env.production`
5. Задеплоит бота и панель управления

В конце выведет:
```

║        🌐 ПАНЕЛЬ УПРАВЛЕНИЯ              ║
╠══════════════════════════════════════════╣
║  URL:     http://YOUR_IP:8742            ║
║  Логин:   admin                          ║
║  Пароль:  xK7mP2nQ9rL4vY6w              ║
╚══════════════════════════════════════════╝
```

### 4. Получение Telegram Session String (опционально)

Нужен только для чтения входящих переписок. Запусти один раз на своей машине:

```bash
./scripts/get-telegram-session.sh
```

Скрипт авторизует тебя через Docker (ничего не нужно устанавливать дополнительно) и выдаст строку для вставки в панель управления.

## Повторный деплой

При обновлении кода или настроек запусти:

```bash
./deploy.sh --kira-mind-bot --server-ip YOUR_VPS_IP
```

Учётные данные панели управления сохраняются между деплоями — повторно генерируются только если не существуют.

## Переменные окружения

Все переменные управляются через веб-панель. Если нужна ручная настройка — скопируй `.env.template` в `.env.production` и заполни.

### Обязательные

| Переменная | Описание |
|------------|----------|
| `OPENAI_API_KEY` | Ключ OpenAI |
| `KIRA_BOT_TOKEN` | Токен Telegram-бота |
| `KIRA_ALLOWED_USER_ID` | Твой Telegram User ID |
| `DB_PASSWORD` | Пароль PostgreSQL |

### Telegram User Client

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_API_ID` | API ID из my.telegram.org |
| `TELEGRAM_API_HASH` | API Hash |
| `TELEGRAM_SESSION_STRING` | Строка сессии (получить через `./scripts/get-telegram-session.sh`) |

### Опциональные интеграции

| Переменная | Описание |
|------------|----------|
| `GOOGLE_MAPS_API_KEY` | Карты и маршруты |
| `IDEOGRAM_API_KEY` | Генерация изображений |
| `USER_TIMEZONE` | Часовой пояс (по умолчанию `Europe/Moscow`) |

### Второй бот (Sergey)

| Переменная | Описание |
|------------|----------|
| `SERGEY_BOT_TOKEN` | Токен второго бота |
| `SERGEY_ALLOWED_USER_ID` | Telegram User ID владельца второго бота |

## Архитектура

```
Telegram → grammy Bot API
              ↓
         Orchestrator
         (классификация интента → план → агенты)
              ↓
    ┌─────────┴──────────┐
    │                    │
 Agents              Services
 conversation        Qdrant (память)
 reminder            PostgreSQL (напоминания)
 maps                OpenAI (LLM, STT, embeddings)
 webSearch           Telegram Client (переписки)
 readMessages        Google Maps
 sendMessages        Ideogram
 imageGeneration
    │
 Schedulers (фоновые)
 kiraLife / dmReport / memoryInsight
```

- **Один кодовая база, два профиля** — `ASSISTANT_PROFILE=KiraMindBot` или `SergeyBrainBot`
- **Личность** — редактируется в веб-панели без перекомпиляции (хранится в `personality.json`)
- **Память** — Qdrant с доменной организацией, переживает рестарты через Docker volume
- **Автовосстановление** — все контейнеры запускаются автоматически после перезагрузки VPS (`restart: unless-stopped`)

## Важные замечания

| Тема | Замечание |
|------|-----------|
| Доступ | Только один владелец — `KIRA_ALLOWED_USER_ID` |
| Без Google Maps | Карты недоступны, остальное работает |
| Без Ideogram | Генерация изображений недоступна, остальное работает |
| Без Telegram Client | Чтение переписок недоступно, диалог с ботом работает |
| Групповые чаты | Включи бота в группу, убедись что privacy mode в BotFather выключен |
| Pinecone | Заглушка в коде, используй `VECTOR_PROVIDER=qdrant` |
