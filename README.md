# Kira Mind Bot

Персональный Telegram-ассистент с долговременной памятью, проактивными подсказками и возможностью работать не только в чате бота, но и с вашим Telegram-аккаунтом (чтение/суммаризация входящих, отправка сообщений контактам, анализ переписок).

## Зачем это пользователю

`kira-mind-bot` полезен как один «центр управления» личными и рабочими задачами:

- Помнит важные факты о вас и использует их в диалоге.
- Создает напоминания в чате и может отправлять `.ics` для календаря.
- Экономит время на Telegram: собирает непрочитанные, делает сводки, анализирует диалоги.
- Помогает с бытовыми и городскими задачами: маршруты, места рядом, адреса.
- Умеет общаться в живом стиле, работать с голосом, фото и пересланными тредами.

## Что умеет бот

### Пользовательские сценарии

- Свободный диалог и ответы на вопросы.
- Напоминания: создание, перенос, завершение, просмотр активных.
- Голосовые сообщения: распознавание речи и обработка как текста.
- Изображения: анализ фото/медиагрупп, OCR/описание, генерация картинок по запросу.
- Геолокация и карты: маршруты, поиск мест рядом, данные по адресу.
- Веб-поиск: поиск актуальной информации через OpenAI web search tools.
- Пересланные сообщения/треды: объединение и разбор в контексте.
- Работа с Telegram-аккаунтом:
  - сводка непрочитанных сообщений;
  - анализ переписки с контактом или группой;
  - отправка сообщений и отложенная отправка;
  - режим «переговоров от вашего имени» с уточнениями.

### Память и проактивность

- Долговременная векторная память по доменам (`work`, `family`, `health`, и т.д.).
- Извлечение и сохранение фактов из диалогов.
- Поиск и использование релевантных фактов в ответах.
- Команды управления памятью (статистика, поиск, чистка, инсайты, удаление фактов).
- Проактивные сценарии:
  - `kiraLifeScheduler` — инициативные сообщения от ассистента;
  - `memoryInsightScheduler` — своевременные подсказки на базе памяти;
  - `dmReportScheduler` — периодические отчеты по входящим.

## Технические возможности

- Архитектура оркестратора: классификация интента -> план -> выполнение агентами.
- Набор специализированных агентов: `conversation`, `reminder`, `maps`, `webSearch`, `readMessages`, `sendMessages`, `imageGeneration`, и др.
- Интеграции:
  - OpenAI (`chat`, `responses`, `embeddings`, `audio transcription`);
  - Qdrant (основное хранилище памяти);
  - Telegram Bot API (`grammy`);
  - Telegram user client (`telegram`) для работы с вашим аккаунтом;
  - Google Maps API;
  - Ideogram API для генерации изображений.
- Профили ассистента: `KiraMindBot` и `SergeyBrainBot` через `ASSISTANT_PROFILE`.
- Docker-ready: отдельный `Dockerfile` + запуск через общий `docker-compose.yml` проекта.

## Отличия от похожих проектов

Ниже не «лучше/хуже», а позиционирование по задачам.

### OpenClaw

Сайт: [openclaw.ai](https://openclaw.ai/)

| Аспект | OpenClaw | Kira Mind Bot |
|--------|----------|---------------|
| Фокус | Универсальная платформа «личного AI-оператора», много каналов (WhatsApp, Telegram, и др.) | Узкий Telegram-first ассистент под одного владельца |
| Память и контекст | Персистентная память, интеграции «со всем» | Доменная векторная память (Qdrant), явные команды памяти |
| Telegram | Один из каналов | Ядро: бот + при необходимости user client (переписки, отправка от имени аккаунта) |
| Сложность | Шире поверхность: skills, плагины, полный доступ к машине (по задумке продукта) | Меньше «операционной системы»: Node.js, Docker, предсказуемый набор агентов |
| Когда смотреть OpenClaw | Нужен агент «на всё» с множеством интеграций и каналов | Когда достаточно сильного личного ассистента в Telegram с памятью и бытовыми сценариями |

### Ouroboros

Репозиторий: [joi-lab/ouroboros](https://github.com/joi-lab/ouroboros)

| Аспект | Ouroboros | Kira Mind Bot |
|--------|-----------|---------------|
| Идея | Self-modifying агент: эволюция кода, фоновое «сознание», бюджеты LLM | Стабильный код: оркестратор + фиксированные агенты под задачи пользователя |
| Риск-профиль | Высокая автономность (в т.ч. `/evolve`), исследовательский характер | Предсказуемые сценарии: напоминания, диалог, карты, переписки |
| Стек | Python, Colab, OpenRouter, GitHub, супервизор воркеров | TypeScript, GrammY, OpenAI, Qdrant |
| Когда смотреть Ouroboros | Эксперименты с автономной эволюцией агента и самоизменением репозитория | Повседневный персональный ассистент без режима «агент переписывает себя» |

### Когда выбирать Kira Mind Bot

| Ваш запрос | Подходит |
|------------|----------|
| Один основной пользователь, общение в Telegram | Да |
| Память о фактах, инсайты, напоминания, `.ics` | Да |
| Сводки непрочитанных, анализ чатов, отправка контактам | Да (нужен Telegram user client) |
| Нужен «универсальный оператор на всю ОС» из коробки | Скорее [OpenClaw](https://openclaw.ai/) |
| Нужна автономная эволюция кода агента | Скорее [Ouroboros](https://github.com/joi-lab/ouroboros) |

## Переменные окружения

Проект загружает конфиг из `.env.${NODE_ENV}` (например, `.env.development` или `.env.production`).

### Обязательные (минимум для запуска)

| Переменная | Описание |
|------------|----------|
| `NODE_ENV` | `development` или `production` |
| `ASSISTANT_PROFILE` | `KiraMindBot` или `SergeyBrainBot` |
| `OPENAI_API_KEY` | Ключ OpenAI |
| `KIRA_BOT_TOKEN` | Токен бота для профиля `KiraMindBot` |
| `SERGEY_BOT_TOKEN` | Токен бота для профиля `SergeyBrainBot` |

### Память (Qdrant)

| Переменная | Описание |
|------------|----------|
| `VECTOR_PROVIDER` | Рабочий вариант: `qdrant` (значение по умолчанию) |
| `QDRANT_URL` | URL Qdrant, по умолчанию `http://localhost:6333` |
| `QDRANT_API_KEY` | Если инстанс Qdrant защищён ключом |
| `VECTOR_SEARCH_THRESHOLD` | Порог релевантности векторного поиска (0–1), опционально |

### Telegram user client (переписки, отправка от аккаунта)

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_API_ID` | API ID приложения Telegram |
| `TELEGRAM_API_HASH` | API Hash |
| `TELEGRAM_SESSION_STRING` | Строка сессии авторизованного пользователя |

Без этих трёх переменных недоступны чтение/анализ переписок и отправка сообщений через ваш аккаунт (не через Bot API).

### Опциональные интеграции

| Переменная | Описание |
|------------|----------|
| `GOOGLE_MAPS_API_KEY` | Карты, маршруты, поиск мест рядом |
| `IDEOGRAM_API_KEY` | Генерация изображений (Ideogram) |
| `USER_TIMEZONE` | Таймзона, по умолчанию `Europe/Moscow` |
| `REMINDER_EXPIRY_TIME_MS` | Срок жизни отложенных reminder intents |

### Проактивные и фоновые настройки

| Переменная | Описание |
|------------|----------|
| `KIRA_PROACTIVE_ENABLED` | Проактивные сообщения (профиль Кира) |
| `KIRA_PROACTIVE_INTERVAL_MS` | Интервал между проактивными сообщениями |
| `KIRA_PROACTIVE_QUIET_HOURS_ENABLED` | Тихие часы для проактивки |
| `KIRA_PROACTIVE_QUIET_HOUR_START` / `KIRA_PROACTIVE_QUIET_HOUR_END` | Границы тихих часов (часы суток) |
| `SERGEY_PROACTIVE_ENABLED` | То же для профиля Сергей |
| `SERGEY_PROACTIVE_INTERVAL_MS` | Интервал (Сергей) |
| `SERGEY_PROACTIVE_QUIET_HOURS_ENABLED` | Тихие часы (Сергей) |
| `SERGEY_PROACTIVE_QUIET_HOUR_START` / `SERGEY_PROACTIVE_QUIET_HOUR_END` | Границы (Сергей) |
| `DM_REPORT_ENABLED` | Периодические отчёты по входящим |
| `DM_REPORT_INTERVAL_MS` | Интервал отчётов |
| `DM_REPORT_QUIET_HOURS_ENABLED` | Тихие часы для отчётов |
| `MEMORY_INSIGHT_ENABLED` | Проактивные подсказки на основе памяти |

## Локальный запуск

```bash
cd kira-mind-bot
npm install
```

1. Создайте файл окружения (например, `.env.development`) и заполните обязательные переменные.
2. Убедитесь, что Qdrant доступен.
3. Запустите бота:

```bash
npm run start:dev
```

### Production-режим локально

```bash
npm run build
npm run start:prod
```

## Запуск через Docker Compose (рекомендуется)

Из корня репозитория:

```bash
docker compose up -d qdrant kira-mind-bot
```

Для второго профиля ассистента:

```bash
docker compose up -d sergey-brain-bot
```

Compose уже настроен на использование `./kira-mind-bot/.env.production`.

## Полезные команды в чате бота

| Команда | Назначение |
|---------|------------|
| `/help` | Список возможностей |
| `/reminders` | Активные напоминания |
| `/history` | Недавняя история диалога |
| `/clear` | Очистка истории (с попыткой сохранить факты в память) |
| `/summary` | Текущая суммаризация диалога |
| `/telegram_unread` | Непрочитанные за 24 часа |
| `/telegram_read` | Отметить все сообщения как прочитанные |
| `/telegram_reset` | Сброс хранилища сообщений Telegram в памяти бота |
| `/contacts` | Список контактов |
| `/admin_menu` | Админ-меню памяти и диагностики (только для `adminUserId`) |

Остальные команды памяти (`/memory_stats`, `/memory_search`, …) и фразы вроде «что ты знаешь обо мне» перечислены после входа в `/admin_menu` или в ответах бота.

## Важные ограничения и замечания

| Тема | Замечание |
|------|-----------|
| Доступ | Персональный ассистент: `allowedUserId` задаётся в `config.ts` |
| Векторная БД | Pinecone в коде — заглушка; используйте `VECTOR_PROVIDER=qdrant` |
| Telegram + контакты | Нужны `TELEGRAM_*` и валидная сессия user client |
| Опциональные API | Без `GOOGLE_MAPS_API_KEY` / `IDEOGRAM_API_KEY` часть функций отключена, остальное работает |
